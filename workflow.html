1

Claude.ai と Claude Code の役割分担

制作作業は 「考える」 と 「手を動かす」 の2つに分かれる。それぞれ担当ツールを固定することで効率よく進める。

Claude.ai 考える・相談する
  • 案件の初回ヒアリング
  • ページ構成・情報設計の検討
  • コンテンツ内容の整理と提案
  • ワイヤーフレームのレビュー・改善提案
  • クライアントフィードバックの整理・優先順位付け
  • 制作方針の相談・判断
Claude Code 手を動かす
  • ワイヤーフレームHTML の全ページ生成
  • ページ間ナビゲーションリンクの設定
  • CLAUDE.md の作成・更新
  • Figma へのワイヤーフレーム書き出し
  • Figma からの差分読み取りと HTML 反映
  • フォルダ・ファイル管理全般
原則： 手を動かす作業はすべて Claude Code が担当。
Claude.ai は成果物を確認・レビューし、次の指示を出す役割。両者を混同しない。
2

基本仕様（全案件共通）

以下は wireframe-projects/CLAUDE.md に定義された全案件共通ルール。

項目仕様
制作物 ワイヤーフレーム（HTML ファイル）
カラー グレースケールのみ使用（カラーコードは #222〜#fff の無彩色）
レスポンシブ 必須。PC / タブレット / モバイルの 3 段階で対応
コンテンツ幅 max-width: 1200px　センタリング
左右パディング PC: padding: 0 40px　／　モバイル: padding: 0 16px
ブレークポイント PC：1024px 以上
タブレット：600px〜1023px
モバイル：599px 以下
ナビゲーション タブレット以下の表示方法は案件ごとの CLAUDE.md で定義（例：ハンバーガーメニュー）
ページ間リンク HTML リンクで接続し、ブラウザ上でプロトタイプとして動作させる
ファイル管理 案件ごとにサブフォルダを作成。各案件フォルダに CLAUDE.md を配置
3

新規案件の開始手順

新しい案件が発生したときの標準的な進め方。

1
Claude.ai でヒアリング・ページ構成の検討
クライアントの要件を Claude.ai に伝え、必要なページ一覧、各ページの役割、ナビゲーション構成などを整理する。
  • どんなページが必要か？（トップ・一覧・詳細 etc.）
  • タブレット以下のナビゲーション方式は？（ハンバーガー、タブ等）
  • Figma への書き出しが必要なページはどれか？
    • 2
    Claude Code で案件フォルダを作成
    wireframe-projects/ 直下に案件名のフォルダを作成する。
    3
    Claude Code で案件固有の CLAUDE.md を作成
    以下の内容を含む CLAUDE.md を案件フォルダ内に作成するよう Claude Code に指示する。
  • 案件名
  • ページ一覧（ファイル名：ページ名）
  • ナビゲーション方式（タブレット以下）
  • Figma 連携情報（チームキー・ファイルキー）※必要な場合
  • 備考（特記事項・制約など）
    • 4
    Claude Code で全ページの HTML を生成
    共通ルール（グレースケール・1200px・レスポンシブ）に沿ったワイヤーフレーム HTML をダミーコンテンツで全ページ生成する。
  • 全ページにヘッダーナビを設置し、相互リンクで接続
  • 現在ページをアクティブ表示（下線・太字等）
  • フッターにも全ページリンクを設置
    • 5
    Claude.ai でレビュー・修正指示
    生成された HTML をブラウザで確認し、不足・修正点があれば Claude.ai を通じてまとめ、Claude Code に修正を依頼する。
    4

    Figma 連携手順

    特定ページを Figma に書き出す場合、または Figma 上の変更を HTML に反映する場合の手順。

    前提： 案件フォルダの CLAUDE.md に Figma ファイルキー・チームキーが記載されていること。Claude Code は自動的に CLAUDE.md を参照して使用する。
    ▼ HTML → Figma への書き出し
    1
    対象の HTML ファイルを Claude Code に読み込ませる
    「○○.html の内容を読み込んで、Figma の Page 1 に書き出してください」と指示する。
    2
    Claude Code が Figma MCP ツールで書き出しを実行
    get_metadata でページ構造を確認後、use_figma でフレームを生成する。グレースケール・1440px 幅で作成される。
    3
    Figma 上で確認・追加編集（手動）
    書き出されたフレームを Figma アプリで開き、必要に応じて画像の追加・テキストの調整などを行う。
    ▼ Figma → HTML への反映
    1
    Claude Code に Figma の読み取りを指示
    「Figma の Page 1 にある○○フレームの内容を読み取って、○○.html に反映してください」と指示する。
    2
    Claude Code が差分を確認して HTML を更新
    get_metadata → get_design_context の順に取得し、Figma 上の変更点のみを HTML に反映する。
    注意： Figma MCP で取得できる画像アセット URL は 7日間 で期限切れになる。URL をそのまま HTML に使う場合は次の「画像のローカル保存手順」を必ず実施すること。
    5

    画像のローカル保存手順

    Figma から取得した一時 URL の画像をローカルに保存し、HTML から参照する手順。

    1
    Claude Code に images フォルダの作成を指示
    案件フォルダ直下に images/ フォルダを作成するよう指示する。
    // 指示例 「しなの鉄道フォルダ内に images フォルダを作成してください。 画像ファイルは手動で配置します。」
    2
    Figma の一時 URL から画像を手動ダウンロード
    Claude Code が HTML に出力した https://www.figma.com/api/mcp/asset/... の URL をブラウザで開き、画像をダウンロードする。
  • ダウンロードしたファイルを images/ フォルダに移動・リネーム
  • ファイル名は用途がわかる名前にする（例：hero.jpg、banner_suica.jpg）
    • 3
    Claude Code に src の書き換えを指示
    一時 URL からローカルパスに変更するよう指示する。
    // 指示例 「top.html のヒーロー画像を Figma の一時 URL から ローカルの images/hero.jpg に変更してください。」
    推奨ファイル命名規則： hero.jpg（ヒーロー画像）、pickup_01.jpg（連番バナー）など、役割がわかる名前にする。
    6

    クライアントフィードバック対応

    クライアントからの修正依頼を受けてから反映するまでの流れ。

    1
    Claude.ai でフィードバックを整理・分類
    受け取ったフィードバックを Claude.ai に共有し、修正の優先順位・対応方法・影響範囲を整理する。
  • 全ページ共通の変更か、特定ページのみか
  • レイアウト変更か、テキスト修正か、構造変更か
  • Figma への再書き出しが必要か
    • 2
    Claude Code に修正を指示
    Claude.ai でまとめた修正内容を Claude Code に渡し、HTML ファイルを更新する。複数ページにわたる共通変更（ナビ構造等）は一括で依頼する。
    3
    ブラウザで動作確認
    修正後のHTMLをブラウザで開き、PC / タブレット / モバイルの各サイズで表示を確認する。ページ間リンクが正常に機能しているかもチェック。
    4
    必要に応じて Figma を更新
    Figma 上でデザインレビューしている場合は、HTML の変更を Figma にも反映する。「○○.html の変更内容を Figma の既存フレームに反映してください」と指示する。
    5
    クライアントへの提出
    修正済みファイル一式（HTMLフォルダ）を圧縮して提出、またはブラウザ確認用 URL を共有する。
    R

    フォルダ構成

    wireframe-projects/ ├── CLAUDE.md # 全案件共通ルール ├── workflow.html # 本ドキュメント │ ├── 案件名フォルダ/ │ ├── CLAUDE.md # 案件固有の設定 │ ├── top.html # トップページ │ ├── page2.html # 各ページ │ └── images/ # 画像ファイル置き場 │ └── hero.jpg # ローカル保存画像 │ └── 別案件フォルダ/ └── ...
    R

    案件 CLAUDE.md の記載仕様

    各案件フォルダの CLAUDE.md に含める項目。Claude Code がこのファイルを自動参照して作業を進める。

    # 案件名 ○○会社 ○○ページ ## ページ一覧 - top.html: トップページ - about.html: 会社概要ページ ## ナビゲーション - タブレット以下でハンバーガーメニューに切り替え ## Figma連携 - チームキー: team::XXXXXXXXXXXXX（チーム名） - ファイルキー: XXXXXXXXXXXXX（ファイル名） ## 備考 - 特記事項・制約・未確定事項など
    項目説明
    案件名クライアント名とページ・サービス名
    ページ一覧ファイル名：ページ名の形式で全ページを列挙
    ナビゲーションタブレット以下の表示方式（ハンバーガー等）
    Figma連携チームキーとファイルキー。Figma不要の案件は省略可
    備考アコーディオン使用、画像入稿予定等の特記事項
