1

ツールと役割分担

制作作業は 「考える」 と 「手を動かす」 の2つに分かれる。それぞれ担当ツールを固定することで効率よく進める。

Claude.ai 考える・相談する
  • 案件の初回ヒアリング
  • ページ構成・情報設計の検討
  • コンテンツ内容の整理と提案
  • ワイヤーフレームのレビュー・改善提案
  • クライアントフィードバックの整理・優先順位付け
  • 制作方針の相談・判断
Claude Code 手を動かす
  • ワイヤーフレームHTMLの全ページ生成
  • ページ間ナビゲーションリンクの設定
  • CLAUDE.mdの作成・更新
  • Figmaへのワイヤーフレーム書き出し
  • Figmaからの差分読み取りとHTML反映
  • git commit / push（GitHub連携）
原則： 手を動かす作業はすべて Claude Code が担当。
Claude.ai は成果物を確認・レビューし、次の指示を出す役割。両者を混同しない。
2

フォルダ構成

すべての案件は wireframe-projects/ 以下に案件フォルダとして管理する。

wireframe-projects/ ├── CLAUDE.md # 全案件共通ルール ├── workflow.html # 本ドキュメント │ ├── 案件名フォルダ/ │ ├── CLAUDE.md # 案件固有の設定 │ ├── top.html # トップページ │ ├── page2.html # 各ページ │ └── images/ # 画像ファイル置き場（必要な場合） │ └── hero.jpg │ └── 別案件フォルダ/ └── ...
ファイル名のルール： 英数字・ハイフンのみ使用。日本語・スペース禁止。
例：marketing-top.html、about.html、service-detail.html

新規案件の立ち上げ手順

新しい案件が発生したときの標準的な進め方。

1
Claude.ai でヒアリング・ページ構成を整理する
クライアントの要件を Claude.ai に伝え、必要なページ一覧・ナビゲーション構成・Figma連携の要否などを整理する。
  • どんなページが必要か？（トップ・一覧・詳細 etc.）
  • タブレット以下のナビゲーション方式は？（ハンバーガー等）
  • Figma への書き出しが必要なページはどれか？
    • 2
    Claude Code で案件フォルダと CLAUDE.md を作成する
    wireframe-projects/ 直下に案件名のフォルダを作成し、案件固有の CLAUDE.md を配置するよう Claude Code に指示する。
  • 案件名・ページ一覧・ナビゲーション方式・Figma情報（必要時）を記載
    • 3
    Claude Code で全ページの HTML を生成する
    共通ルール（グレースケール・1200px・レスポンシブ）に沿ったワイヤーフレーム HTML をダミーコンテンツで全ページ生成する。
  • 全ページにヘッダーナビを設置し、相互リンクで接続
  • フッターにも全ページリンクを設置
    • # ターミナルで Claude Code を起動する claude
    4
    ブラウザで確認する
    生成された HTML をブラウザで開き、PC / タブレット / モバイルの各サイズで表示とページ間リンクを確認する。
    # ターミナルから直接ブラウザで開く open wireframe-projects/案件フォルダ/top.html
    4

    修正・更新の手順

    既存のワイヤーフレームを修正・更新するときの流れ。GitHub push 後は Netlify が自動デプロイする。

    1
    ブラウザで現状を確認する
    修正前に現在の表示を確認し、変更が必要な箇所を把握する。
    2
    Claude Code に修正を指示する
    変更内容を Claude Code に伝え、HTML ファイルを更新する。複数ページにわたる共通変更（ナビ構造等）は一括で依頼する。
    3
    ブラウザで修正を確認する
    PC / タブレット / モバイルの各サイズで表示を確認する。ページ間リンクが正常に機能しているかもチェックする。
    4
    git commit & push する
    変更内容をコミットして GitHub に push する。以下のワンライナーで一括実行できる。
    git -C ~/Documents/wireframe-projects add . && \ git -C ~/Documents/wireframe-projects commit -m "修正内容のメモ" && \ git -C ~/Documents/wireframe-projects push
    5
    Netlify が自動デプロイする
    push を検知して Netlify が自動デプロイを開始する。通常 1〜2 分でクライアント共有 URL に反映される。
  • Netlify ダッシュボードの「Deploys」タブでデプロイ状況を確認できる
  • デプロイ完了後、共有 URL をクライアントに案内する
    • 5

    GitHub・Netlify 連携設定（初回のみ）

    リポジトリを GitHub に登録し、Netlify で自動デプロイを有効にするまでの初期設定。一度設定すれば以降は不要。

    ▼ GitHub リポジトリの初期設定
    1
    ローカルで git を初期化して GitHub に push する
    wireframe-projects/ フォルダをリポジトリとして登録する。
    cd ~/Documents/wireframe-projects git init git add . git commit -m "first commit" git branch -M main git remote add origin https://github.com/ユーザー名/wireframe-projects.git git push -u origin main
    2
    GitHub Personal Access Token を発行する
    push に認証が必要な場合は GitHub の Settings → Developer settings → Personal access tokens から Token を発行し、push 時の URL に埋め込む。
    git remote set-url origin https://ユーザー名:TOKEN@github.com/ユーザー名/wireframe-projects.git
    注意： Token は定期的にローテーションすること。コードや履歴にそのまま残さない。
    ▼ Netlify の自動デプロイ設定
    1
    Netlify にログインして新規サイトを作成する
    Netlify ダッシュボードで「Add new site」→「Import an existing project」を選択する。
    2
    GitHub リポジトリを接続する
    「Deploy with GitHub」を選択し、wireframe-projects リポジトリを選ぶ。
    3
    デプロイ設定を行い「Deploy site」を実行する
    以下の設定を確認してからデプロイを開始する。
  • ブランチ：main
  • ビルドコマンド：（空欄のまま）
  • 公開ディレクトリ：/（ルート）
    • 4
    発行された URL をクライアントに共有する
    デプロイ完了後、Netlify が自動生成した URL（例：https://xxx.netlify.app）をクライアントに案内する。以降は push のたびに自動更新される。
    6

    Figma 連携手順

    特定ページを Figma に書き出す場合、または Figma 上の変更を HTML に反映する場合の手順。

    前提： 案件フォルダの CLAUDE.md に Figma ファイルキー・チームキーが記載されていること。Claude Code は自動的に CLAUDE.md を参照して使用する。
    ▼ HTML → Figma への書き出し
    1
    対象の HTML ファイルを Claude Code に読み込ませる
    「○○.html の内容を読み込んで、Figma の Page 1 に書き出してください」と指示する。
    2
    Claude Code が Figma MCP ツールで書き出しを実行する
    get_metadata でページ構造を確認後、use_figma でフレームを生成する。グレースケール・1440px 幅で作成される。
    3
    Figma 上で確認・追加編集する（手動）
    書き出されたフレームを Figma アプリで開き、必要に応じて画像の追加・テキストの調整などを行う。
    ▼ Figma → HTML への反映
    1
    Claude Code に Figma の読み取りを指示する
    「Figma の Page 1 にある○○フレームの内容を読み取って、○○.html に反映してください」と指示する。
    2
    Claude Code が差分を確認して HTML を更新する
    get_metadata → get_design_context の順に取得し、Figma 上の変更点のみを HTML に反映する。
    注意： Figma MCP で取得できる画像アセット URL は 7日間 で期限切れになる。URL をそのまま HTML に使う場合は、画像を images/ フォルダにダウンロードしてローカルパスに変更すること。
    7

    クライアントフィードバック対応

    クライアントからの修正依頼を受けてから反映するまでの流れ。

    1
    Claude.ai でフィードバックを整理・分類する
    受け取ったフィードバックを Claude.ai に共有し、修正の優先順位・対応方法・影響範囲を整理する。
  • 全ページ共通の変更か、特定ページのみか
  • レイアウト変更か、テキスト修正か、構造変更か
  • Figma への再書き出しが必要か
    • 2
    Claude Code に修正を指示する
    Claude.ai でまとめた修正内容を Claude Code に渡し、HTML ファイルを更新する。複数ページにわたる共通変更（ナビ構造等）は一括で依頼する。
    3
    ブラウザで動作確認する
    修正後の HTML をブラウザで開き、PC / タブレット / モバイルの各サイズで表示を確認する。ページ間リンクが正常に機能しているかもチェックする。
    4
    git push して Netlify に自動反映させる
    「4. 修正・更新の手順」のワンライナーで commit & push する。Netlify が自動デプロイし、クライアント共有 URL が更新される。
    5
    必要に応じて Figma を更新する
    Figma 上でデザインレビューしている場合は、HTML の変更を Figma にも反映する。「○○.html の変更内容を Figma の既存フレームに反映してください」と指示する。
    R

    案件 CLAUDE.md の記載仕様

    各案件フォルダの CLAUDE.md に含める項目。Claude Code がこのファイルを自動参照して作業を進める。

    # 案件名 ○○会社 ○○ページ ## ページ一覧 - top.html: トップページ - about.html: 会社概要ページ ## ナビゲーション - タブレット以下: ハンバーガーメニュー - アイコン: Material Icons（`menu` で開く / `close` で閉じる） - 展開時: 全画面オーバーレイ・縦並びリスト ## ナビ項目 1. ○○ 2. ○○ 3. ○○（CTAボタン） ## Figma連携 - チームキー: team::XXXXXXXXXXXXX（チーム名） - ファイルキー: XXXXXXXXXXXXX（ファイル名） ## 備考 - 特記事項・制約・未確定事項など
    項目説明
    案件名 クライアント名とページ・サービス名
    ページ一覧 ファイル名：ページ名の形式で全ページを列挙
    ナビゲーション タブレット以下の表示方式を記載。ハンバーガーメニューの場合は以下を明記する：
    ・アイコン（Material Icons の menu / close）
    ・展開方式（全画面オーバーレイ・縦並びリスト等）
    ナビ項目 ナビゲーションに表示する項目を順番に列挙。CTAボタンも含める
    Figma連携 チームキーとファイルキー。Figma不要の案件は省略可
    備考 ドロップダウン使用、画像入稿予定、未確定事項等の特記事項
