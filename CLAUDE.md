# wireframe-projects

## 制作物
- ワイヤーフレーム（HTMLファイル）
- グレースケールのみ使用
- レスポンシブ対応必須

## レイアウト
- コンテンツ幅: 1200px
- 左右パディング: 40px（モバイル: 16px）

## ブレークポイント
- PC: 1024px以上
- タブレット: 600px〜1023px
- モバイル: 599px以下

## ファイル管理
- 案件ごとにサブフォルダを作成する
- 各案件フォルダにも案件固有のCLAUDE.mdを作成する
- ページ間はHTMLリンクで接続してプロトタイプとして機能させる
- ファイル名は英数字・ハイフンのみ（日本語・スペース禁止）

## HTMLの基本構造
- 文字コード: `<meta charset="UTF-8">`
- viewport: `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
- スタイルはすべてCSSクラスで定義する（インラインスタイル禁止）
- 同一要素に同じプロパティを持つクラスを複数付与しない

## 外部リソース
- フォント: `<link href="https://fonts.googleapis.com/css2?family=Noto+Sans+JP:wght@400;500;700&display=swap" rel="stylesheet">`
- アイコン: `<link href="https://fonts.googleapis.com/icon?family=Material+Icons" rel="stylesheet">`

## カラー定義
- 黒: #333333
- 白: #ffffff
- ダークグレー: #888888（サブテキスト・注釈・プレースホルダー）
- ライトグレー: #f0f0f0（セクション背景・カード背景）
- ボーダー: #dddddd

## タイポグラフィ
- H1: 36px / bold / 行間130%（モバイル: 28px）
- H2: 30px / bold / 行間130%（モバイル: 28px）
- H3: 26px / bold / 行間130%（モバイル: 24px）
- H4: 22px / bold / 行間130%（モバイル: 22px）
- H5: 18px / bold / 行間130%（モバイル: 18px）
- 本文p: 16px / regular / 行間200%
- 注釈・キャプション: 12px / regular / 行間200%

## H1の使い方
- ページに必ず1つだけ配置する
- トップページ: ヒーローセクションのキャッチコピーをH1にする
- 下層ページ: ページタイトルをH1にする
- H1はページの主題を表すため、ナビゲーションやロゴには使用しない

## 余白
### セクション間
- 白背景セクション（PC）: 上下100px / タブレット: 60px / モバイル: 40px
- 色で区切るセクション（PC）: 上下50px / タブレット: 30px / モバイル: 20px

### 見出しの余白
- セクションの先頭に見出しがある場合、見出しの上余白は0px（セクション間の余白を優先）
- それ以外の場合は以下の余白を適用する

| 見出し | 上余白（PC） | 上余白（SP） | 下余白（PC） | 下余白（SP） |
|---|---|---|---|---|
| H2 | 100px | 80px | 32px | 24px |
| H3 | 60px | 40px | 20px | 20px |
| H4 | 40px | 40px | 16px | 16px |
| H5 | 32px | 32px | 10px | 10px |

## コンポーネント定義

### ヘッダー
- 左: ロゴ＋会社名 / 中央: ナビゲーション（6〜8項目） / 右: CTAボタン
- タブレット以下のナビゲーション表示方法は案件ごとのCLAUDE.mdで定義

### 見出し
- 左ボーダー4px（#333333）＋H2のみ。サブテキストは付けない
- 番号は不要

### カード
- 構成: 画像（3:2）＋見出し＋説明文
- 背景: #f0f0f0・ボーダー1px #dddddd・角丸4px

### ボタン
- 強調: 背景#333333・テキスト#ffffff・角丸4px
- 通常: 背景なし・枠線1px #333333・テキスト#333333・角丸4px

### フッター
- 左: ロゴ＋会社名＋住所・電話番号 / 右: ページ動線（第2〜3階層含む）
- 最下部: コピーライト（背景#f0f0f0）

### 画像プレースホルダー
- メインビジュアル以外は3:2の比率で統一
- プレースホルダーのテキストは「（画像）」で統一

## 実装の注意点

### 全幅背景の上にコンテンツ幅で要素を配置する場合
全幅の背景（メインビジュアル、色付き帯など）の上に、コンテンツ幅(1200px)に揃えたテキストや要素を乗せる構成では、以下に注意する。

- 背景を覆うオーバーレイを position:absolute + display:flex にした場合、その中に置く .container（max-width:1200px, margin:0 auto）は、flexの左右autoマージンの影響で「中身の幅」まで縮み、中央に寄ってしまう
- これを防ぐため、flexコンテナ内の .container には必ず width:100% を指定する
- 内側の要素に width:fit-content を付けない（縮みの原因になる）
- 目的は、背景の上に乗せた要素の左端を、他セクションの .container 左端と完全に一致させること

### 縦積み（flex-direction: column）になったflex子要素の幅
スマホ対応でflex親を `flex-direction: column` に切り替えるとき、子要素が自動的に `width: 100%` になるとは限らない。以下の条件が1つでも当てはまると、子要素は縮んだまま縦に並ぶ。

- 子要素に `flex-shrink: 0` または `flex: none` が付いている
- 子要素またはその子孫に固定幅（`width: 200px` など）が付いている
- 親に `align-items: flex-start` が付いている（デフォルトは `stretch` なので意図せず設定されていることがある）

対処: スマホ用メディアクエリで、縦積みになった子要素に `width: 100%` と `flex: none` を明示する。

### PCで横並びの要素は、スマホ縦積み時に幅指定を必ず見直す
PCで `display: flex` の横並びになっている要素をスマホで縦積みに切り替える際、以下の手順で幅を確認する。

1. 子要素に固定幅・`flex-shrink: 0`・`flex: none` が付いていないか確認する
2. 親に `align-items` の指定がないか確認する
3. 縦積みにした子要素には `width: 100%; flex: none;` を明示する
4. 孫要素に固定幅がある場合は、そちらも `width: 100%; max-width: none;` で上書きする

### PC用の改行（br）はスマホで無効化する
PC表示用に `<br>` タグでテキストを折り返している箇所は、スマホで意図しない位置で改行される原因になる。スマホ用メディアクエリに以下を追加する。

```css
/* 対象要素を絞って指定する */
.section-desc br { display: none; }
```

### 修正指示を受けたら、必ず該当箇所のコードを確認してから直す
「〇〇が直っていない」という修正指示を受けたとき、原因を推測だけで判断せずに、まず該当箇所のHTMLとCSSを読んで現在の状態を確認する。確認なしに修正すると、本当の原因とは別の箇所を変更してしまい、問題が解決しないまま別の不具合を生む。
